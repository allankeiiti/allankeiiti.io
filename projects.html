Ferramenta do Suporte N1

Projeto Solo | Java | 2019
Esse projeto foi desenvolvido exclusivamente para a equipe de suporte da M.L. Gomes, utilizando Java, Java Swing e batch Script para montar a interface gráfica e utilizando SQLite. O projeto foca em agilizar o atendimento dos chamados que o Suporte recebia, tendo a funcionalidade de instalar softwares, fechar programas e tarefas ou executar outros comandos remotamente. A Ferramenta do Suporte N1 foi desenvolvido logo após eu finalizar o Script de Compactador de Arquivos e estipulei um desafio de termina-lo em menos de 1 semana, e sua primeira versão foi concluída dentro do prazo.

Auto Compactador de arquivos

Projeto Solo | Python | 2019
Enquanto estive na equipe de suporte da empresa M.L. Gomes, fui solicitado para desenvolver um script para a equipe de infraestrutura, que dado a quantidade de dias que um arquivo não era acessado, é gravado uma cópia dele em outro diretório e então ele é compactado junto com os outros arquivos que também não foram acessados, seguindo a mesma estrutura de pastas. Terminado a compactação, é gerado um arquivo log da operação, apontando quais foram os arquivos listados, o total de bytes, kbytes, mbytes, gigabytes (Mudando a grandeza conforme a quantidade) e um gráfico de barras com a quantidades de arquivos gravados e compactados por extensão. Foi meu primeiro contato mais a fundo com Python e com a biblioteca MatplotLib. Infelizmente o Script não foi utilizado pela equipe de infraestrutura no final, mas foi essencial para meu aprendizado e foi meu primeiro projeto criado para uma organização.

App de Cadastro

Projeto Solo | Java Android | 2018
Um aplicativo básico que realiza as funcionalidades de Create, Read, Update, Delete, persistindo os dados registrados no dispositivo móvel utilizando o SQLite. Desenvolvi ele durante o 6º Semestre do curso de Bacharelado de Sistemas de Informação, ele foi um marco para mim pois na época fiquei um tempo longe do desenvolvimento, o curso de Android não tinha videos onde podia me ajudar no problema e eu debugava a aplicação printando os valores das variáveis no console ao invés de utilizar a opção de debug da própria IDE e ele foi utilizado como base do TCC de conclusão do meu grupo.
